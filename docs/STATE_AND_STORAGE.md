# BIT101-iOS 状态与存储说明

这份文档专门说明项目里的“状态到底存在哪里”。

当你遇到下面这些问题时，应该先看这份文档：

- 为什么切号后有些状态跟着变，有些不变
- 为什么小组件拿到的数据和主 App 不完全一样
- 某个值到底在 `UserDefaults`、Keychain、文件缓存还是共享容器里
- 改动某个偏好后，为什么另一个页面没有立刻生效

## 1. 状态分类总览

当前项目里的状态大致分成六类：

1. 会话与凭据
2. 模块业务缓存
3. 界面偏好
4. 账号隔离偏好
5. 扩展共享快照
6. 页面级瞬时状态

## 2. 会话与凭据

### 2.1 保存内容

这里主要包括：

- 学校统一认证账号
- 学校统一认证密码
- 与登录恢复相关的敏感凭据
- BIT101 侧会话恢复所需信息

### 2.2 保存位置

这类信息应优先放在：

- Keychain

相关代码主要在：

- `Login/LoginService.swift`

### 2.3 维护原则

- 任何敏感信息不要回落到普通 `UserDefaults`
- 如果登录链路调整了字段名或恢复逻辑，先确认旧 Keychain 迁移是否还兼容

## 3. 课表、DDL、考试与自定义日程

### 3.1 保存内容

- 课程表
- 考试
- DDL
- 自定义日程
- 一些查询所需的本地上下文

### 3.2 保存位置

当前主要是：

- 本地缓存文件

相关代码主要在：

- `Schedule/ScheduleModels.swift`
- `Schedule/ScheduleViewModel.swift`

### 3.3 特点

- 这类数据强调“先恢复缓存，再同步远端”
- 属于账号隔离数据
- 切号后不应继续沿用前一个账号的课表

## 4. 设置与偏好

### 4.1 保存内容

包括但不限于：

- 外观模式
- 自动旋转
- 话廊屏蔽偏好
- 搜索结果是否隐藏机器人
- 灵动岛提前显示阈值
- 一部分查询筛选结果

### 4.2 保存位置

当前主要是：

- `UserDefaults`
- 通过 `AppSettingsStore` 做统一读写

相关代码主要在：

- `Settings/AppSettingsStore.swift`

### 4.3 两类偏好的区别

#### 4.3.1 全局偏好

例如：

- 主题模式
- 屏幕旋转

这类偏好不需要随账号变化。

#### 4.3.2 账号隔离偏好

例如：

- 社区规则接受状态
- 隐藏用户 / 隐藏帖子
- 一部分查询筛选

这类偏好必须按账号分桶。

## 5. 话廊的本地状态

### 5.1 服务端状态

服务端提供：

- 帖子列表
- 评论
- 分类未读数

### 5.2 本地派生状态

客户端额外维护：

- 当前 feed 的分页状态
- 搜索状态
- 本地屏蔽
- 消息页“伪新消息”

这里最容易混淆的是“消息已读”。

当前项目并没有真正的服务端逐条已读状态，而是：

- 服务端给分类未读数
- 客户端基于这份计数做本地近似展示

因此它的本质是：

- UI 体验状态
- 不是严格一致的消息存档状态

## 6. 小组件与锁屏组件的数据来源

### 6.1 为什么不能直接读主 App 缓存

因为：

- 主 App 缓存结构更复杂
- 扩展 target 不应该深耦合业务内部对象
- 共享边界太宽会让维护成本明显上升

### 6.2 当前做法

当前设计是：

1. 主 App 维护完整课表缓存
2. `ScheduleWidgetSupport.swift` 导出精简快照
3. widget / 锁屏组件 / Live Activity 只读取共享快照

### 6.3 保存位置

共享快照保存在：

- App Group 共享容器

标识为：

- `group.BIT101-dev.BIT101-iOS.shared`

## 7. Live Activity 的额外运行时状态

Live Activity 不是“存一份静态数据就结束”，它还依赖运行时判断：

- 当前是否有课
- 下一节课还有多久
- 当前是否命中提前显示阈值
- 当前账号有没有开启该功能

因此它使用的是：

- 共享快照
- 当前时间
- 本地设置
- 当前运行中的 activity 状态

这也意味着：它的显示结果是动态计算的，不是单纯读一份数据库。

## 8. 头像缓存

### 8.1 当前状态

头像不再只依赖系统默认网络缓存，而是有显式缓存层。

### 8.2 目的

- 避免冷启动时重复下载头像
- 减少列表滚动中的图片抖动
- 提高“我的”“话廊”“消息中心”的稳定性

### 8.3 保存位置

这部分同时依赖：

- 内存缓存
- 磁盘缓存

相关代码在：

- `CachedRemoteImage.swift`

## 9. 页面瞬时状态

不是所有状态都应该落盘。

例如：

- sheet 是否打开
- 当前搜索输入
- 当前正在加载更多
- 当前帖子详情是否展开

这类状态通常应停留在：

- `@State`
- `@StateObject`
- `ViewModel` 的内存属性

不要为了“怕丢”就把瞬时状态写进持久化层，否则只会让行为越来越难解释。

## 10. 判断某个新状态该放哪

如果你准备新增一个状态，建议先按下面顺序判断：

1. 它是不是敏感信息
   - 是：优先 Keychain
2. 它是不是跨启动仍要恢复的业务数据
   - 是：缓存文件或 `UserDefaults`
3. 它是不是应该按账号隔离
   - 是：不要存成全局唯一值
4. 它是不是扩展要读
   - 是：导出精简快照，不要直接共享主缓存
5. 它是不是只影响当前页面一小段时间
   - 是：留在内存，不要落盘

这套判断比“看到哪里方便就往哪里存”更重要。
